泛型

  • 如果一个类的属性或者方法的参数或返回值不确定的时候, 需要使用泛型
  • 或者某个类型是动态的时候, 也需要使用泛型

类或接口的泛型

  • 相当于类的参数
  • 声明方式: 在类或接口的声明的时候, 在类名后面跟上 <>, <> 中书写大写字母就代表声明了这个泛型,

泛型声明完成之后, 就可以在当前类中把泛型当做一个类来使用, 这样当使用这个类时, 泛型传入的是什么类型, 该类里面的所有泛型都会变成这个泛型

public interface Iterator<T> {
    T next();
    void add(T obj);
}

public class ArrayList implements Iterator<String> {
    @Override
    public String next() {
        return null;
    }

    @Override
    public void add(String obj) {

    }
}

可以同时声明多个泛型, 在一个 <> 中使用 , 隔开

方法的泛型

  • 声明方式: 在方法的返回值类型前面使用 <> 声明泛型

在方法中声明的泛型只能在该方法中使用, 例如返回值/参数/方法体, 绝大多数情况下, 方法中的泛型都会在方法参数中使用, 因为需要在参数中确定泛型的具体类型

public static <T> void sort(T[] array, MyComparator<T> comparator) {   // 根据T[] array, 即根据第一个参数确定泛型<T>的具体类型
        for (int i = 0; i < array.length - 1; i++) {
            for (int j = 0; j < array.length - 1 - i; j++) {
                if (comparator.compare(array[j], array[j + 1]) > 0){
                    T temp = array[j];
                    array[j] = array[j + 1];
                    array[j + 1] = temp;
                }
            }
        }
    }

异常

异常会导致程序的中断 异常也是类

java中的所有出错分为两类:

Throwable:

  1. Error 错误 通过一般的代码修改无法处理的, 不能使用try,catch捕获
  2. Exception 异常 可以进行处理的

Exception:

  1. RuntimeException 运行时异常
  2. 其他异常 强制检查型异常

RuntimeException: 算数异常等...

语法格式
try {
    可能会抛出异常的代码
} catch (XxxxException e) {
    解决该异常的代码
}

如果try代码块中没有异常, 所有的catch都不会执行 如果try中某一行代码出现异常, try中的该行以后的代码都不会执行, 检查catch有没有捕获对应的异常, 如果有, 则执行对应的catch代码块, 如果没有, 程序直接中断

catch捕获异常的时候按照先后顺序依次判断, 符合多态规则, 所以书写catch的时候要把子类写前面, catch和finall至少得有一个和try搭配

public class MainException {
    public static void main(String[] args) {
        // 构造方法,
        NullPointerException n = new NullPointerException("去你大爷的空指针");
        //throw n;
        // 打印:去你大爷的空指针
        System.out.println(n.getMessage());
        System.out.println("----------");
        // 在控制台打印该异常的日志
        n.printStackTrace();

        Object o = null;
        try {
            System.out.println("1");    //会输出
            System.out.println(5 / 0);     // 先遇到哪个异常, 就catch到哪个, 后面的异常不做处理
            o.toString();   // 空指针异常     
            System.out.println("2");    // 不会输出
        } catch (NullPointerException e) {
            System.out.println("出现了空指针");
        } catch (ArithmeticException e) {
            System.out.println("算数异常");
            e.printStackTrace();
        } finally {
            // 不管是否捕获到了异常, 该代码块都会执行
            // 一般会在该位置执行一些释放资源的操作
        }
        // 抛出异常 throw
        // 语法:  throw 异常对象
        System.out.println("测试程序是否执行完成");
        div(-1, 5);

        IoTest t = new IoTest();    // IoTest类的test()方法抛出了异常
        try {
            t.test(4);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
    private static int div(int x, int y) {
        // 规定该方法只能计算两个正整数的除法
        if (x <= 0 || y <= 0) {
            throw new NumberNotBeNegativeException("传入的参数必须为正整数, 您传入的 x = " + x + ", y = " + y + ", 不合法");   //自定义(自定义异常)的提示
        }
        return x / y;
    }
}

throws

如果在一个方法中不需要处理该异常的话, 可以在方法声明时使用该关键字将异常向上抛出, 在哪里调用该方法, 就在哪里进行捕获, 也遵循多态原则

public class IoTest {
    public void test(int age) throws Exception {   // 将异常抛出
        if (age < 0) {
            throw new IOException();    // 强制检查型异常, 在编译期必须进行异常捕获
        }
        if (age < 10) {
            throw new NoSuchFieldException();   //遵循多态原则, throws后面的为throw后面的父类
        }
    }
}
输入异常处理
public class ScannerDemo {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int sum = 0;
        for (int i = 0; i < 10; i++) {
            System.out.println("请输入第" + (i + 1) + "个数:");
            try {
            int a = sc.nextInt();
            sum += a;
            } catch (InputMismatchException e) {
                System.out.println("输入错误, 请重新输入数字");
                i--;
                sc = new Scanner(System.in);
            }
        }
        System.out.println(sum);
    }
}
